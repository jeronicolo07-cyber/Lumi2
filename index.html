🔗 Paso 1 — Conectar con n8n
¿Qué hace esto? Esta pantalla es solo la interfaz (vive en GitHub Pages). El "cerebro" de LUMI vive en tu workflow de n8n Cloud. Pegá la URL acá y quedan conectados.

Cómo obtener la URL:
1. Entrá a n8n.cloud → tu instancia
2. Abrí el workflow "LUMI Brain"
3. Hacé clic en el nodo 🎙 LUMI Webhook
4. Copiá la Production URL (empieza con https://)
5. Pegala acá abajo y guardá
⚠ Si la prueba falla con error CORS:
En n8n, abrí el nodo "Respond to Webhook" → Options → Response Headers → agregá estos 3 headers:

• Access-Control-Allow-Origin → *
• Access-Control-Allow-Methods → POST, OPTIONS
• Access-Control-Allow-Headers → Content-Type

Luego desactivá y volvé a activar el workflow. Probá de nuevo.
🎙 Paso 2 — Configurar Voz
Reconocimiento de voz (STT): Usa el micrófono del navegador. Funciona en Chrome y Safari con internet. Requiere que le des permiso al micrófono cuando te lo pida la primera vez.

Voz de LUMI (TTS): Usa las voces instaladas en tu celular. Para una voz más natural en español, instalá el paquete de voces de Google desde Ajustes → Accesibilidad → Texto a voz del dispositivo → Instalar datos de voz.
📲 Paso 3 — Instalar como App
Android (Chrome):
1. Abrí esta URL en Chrome
2. Tocá los 3 puntos (⋮) arriba a la derecha
3. Tocá "Agregar a pantalla de inicio"
4. Confirmá → LUMI aparece como app con ícono

iPhone (Safari):
1. Abrí esta URL en Safari
2. Tocá el botón compartir □↑ abajo
3. Tocá "Agregar a pantalla de inicio"
4. Confirmá → queda como app nativa

Una vez instalada, se abre sin barra del navegador, en pantalla completa.
📱 Paso 4 — Tasker (abrir apps y llamadas)
¿Para qué sirve Tasker? Para que LUMI pueda abrir apps, hacer llamadas y ejecutar acciones directamente en tu celular Android cuando n8n le da la orden.

Instalación:
1. Descargá Tasker de Play Store (~$3.99)
2. Descargá Tailscale de Play Store (gratis) — esto le da una IP fija a tu celular para que n8n lo encuentre desde cualquier red

Configurar Tailscale:
3. Creá cuenta en tailscale.com
4. Instalá Tailscale también en tu VPS/PC donde corre n8n
5. En Tailscale, copiá la IP de tu celular (algo como 100.x.x.x)

Configurar Tasker:
6. Abrí Tasker → Perfiles → "+" → Evento
7. Elegí Plugin → HTTP Request Shortcuts o usá el plugin HTTP Server integrado
8. Configurá el servidor en puerto 1821
9. Creá tareas para cada acción:
  • Si recibe open_app → Acción: Lanzar App
  • Si recibe make_call → Acción: Llamada telefónica

Conectar con n8n:
10. En tu workflow de n8n, en el nodo "📱 Tasker: Open App", cambiá la IP por la IP de Tailscale de tu celular:
  http://100.X.X.X:1821/lumi-command
♿ Paso 5 — Accesibilidad (nivel avanzado)
¿Para qué sirve? El AccessibilityService le permite a LUMI "ver" la pantalla y tocar botones de otras apps — incluso leer notificaciones de WhatsApp o hacer clic en cualquier elemento.

Limitación importante: Esto NO se puede hacer desde una PWA/HTML. Requiere una APK nativa compilada en Android Studio.

Pasos para activarlo (cuando tengas la APK):
1. Instalá la APK de LUMI en tu celular
2. Andá a Ajustes del celular
3. Buscá Accesibilidad
4. Entrá a Servicios instalados o Apps descargadas
5. Buscá "LUMI" en la lista
6. Activalo con el toggle
7. Confirmá el permiso cuando aparezca el aviso

⚡ ¿Querés la APK nativa? Pedímela en el chat y te entrego el código completo de Android Studio con Java/Kotlin listo para compilar.
🐙 Paso 6 — GitHub Actions (automatización)
¿Para qué sirve? Podés hacer que LUMI ejecute scripts en la nube al recibir una orden, sin necesidad de tener una PC encendida.

Cómo conectar LUMI con GitHub Actions:
1. En tu repo → Settings → Secrets → Actions
2. Creá un secret llamado LUMI_TOKEN con un token tuyo
3. En n8n, agregá un nodo HTTP Request que llame a:
  POST https://api.github.com/repos/jeronicolo07-cyber/lumi-asistente/dispatches
  Header: Authorization: Bearer TU_TOKEN
  Body: {"event_type":"lumi-command","client_payload":{"cmd":"..."}}
4. En tu repo creá el archivo .github/workflows/lumi.yml que escuche el evento y ejecute el script

Así LUMI puede hacer git pull, correr scripts de Python, actualizar su propio código, y devolverte el resultado.
🐛 Debug y diagnóstico
Con el debug activado, cada vez que enviás un mensaje vas a ver la URL del webhook, el payload enviado, el código HTTP de respuesta y la respuesta recibida — todo en la barra naranja arriba del chat. Útil para diagnosticar problemas de conexión.
